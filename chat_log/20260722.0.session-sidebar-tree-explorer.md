## Problem/Context
- Sidebar must be per-terminal (inside the dockview terminal panel, right side), not a global dock panel, because it is scoped to that session's cwd + agent. Reuse `<TreeTable>` (`src/treetable.tsx`) — already backs WorktreesPanelV2/favorites/activity/status/rules/meme — not a new component.
- `TerminalPanel` (`reactdock.tsx`) restructured to a `[term-slot | sidebar]` flex row; the `.term-host` xterm node now adopts into a `.term-slot` div instead of the bare host.
- Live cwd resolution needs a new dock hook `onTermCwd` (set by main.ts from `tabMetaById`) to avoid a `reactdock → terminal → reactdock` import cycle.
- e2e harness is **frontend-only**: `playwright.config.ts` runs `pnpm dev` on :4173; headless Chrome; `?e2e` flips `src/reactive/nativeTransport.ts` to resolve every `invoke` from `window.__instantE2eNativeResults[command]` (static, per-command) or undefined. No Rust build, no `src-tauri/target`. Entry pattern = custom HTML (`e2e-paint.html`) + bootstrap tsx (`e2e/paint.tsx`). `mountReactDock` registers `terminal: TerminalPanel`; opening a terminal needs `<div id="panel-pool" hidden></div>` in the entry HTML (openTab appends `.term-host` there before dockview adopts — its absence was the first e2e failure).
- "Don't flood": worktree node_modules are pnpm symlinks (cheap); no Rust target/; screenshots live in `test-results/`. Main is the user's daily-driver dev — leave it clean; all sidebar work stays in the worktree.
- `just check` is unsafe (chains `todos-check` → shells out to `dl`, not installed). Run `corepack pnpm@10.12.4 exec tsc --noEmit` directly. Pre-existing tsc baseline = 3 errors (mdview/index.ts:58 openMarkdownPanel; plugin.test.ts:69 CtxItem.label; preview.ts:38 unused ext). Compare new work against these.
- Local-image proof flow: the `Read` tool uploads a local PNG to the harness CDN (glm), returns a URL; `mcp__4_5v_mcp__analyze_image` then describes it. NSIRD screencapture paths are TCC-blocked for the shell but `Read` can still upload them.
- z.ai GLM model mapping (ccz): opus/sonnet slots → `glm-5.2[1m]`, haiku → `glm-4.7`. Delegate throwaway research to `haiku` to save money.

## Key Insights
- `<TreeTable>` (`src/treetable.tsx`) is the one reusable tree/flat table; props include `columns` (one with `tree:true`), `getRowCanExpand`, `onToggleExpand` (lazy), `cell` per column, `controls`+`filter` (search bar), `toggleOnDoubleClick`. Used by every panel already.
- Harness storage (glm-4.7 research, verified on-disk): claude/ccz = `~/.claude/projects/<enc-cwd>/<uuid>.jsonl`; opencode = sqlite `~/.local/share/opencode/opencode.db` (`session`,`message` tables; turns = `SELECT id,time_created,data FROM message WHERE session_id=? ORDER BY time_created,id`); codex = hybrid sqlite `threads` + `~/.codex/sessions/.../rollout-*.jsonl`; kimi = `~/.kimi-code/sessions/wd_<proj>_<hash>/session_<uuid>/agents/main/wire.jsonl` (multi-agent: agent-0/agent-1/main). `read_ai_messages` Rust already covers claude/opencode/codex; kimi needs a new reader.
- Harness spread to delete: `DEFAULT_WT_AGENTS` (state.ts), `QUICK_CMD` (terminal.ts), quick-launch buttons (tablepanels.tsx), `ai.favTurn` (main.ts) — all miss ccz+kimi.
- FitAddon off-by-one root cause (for context): it reads `getComputedStyle(.term-host).height` = border-box under global `box-sizing:border-box`; `.term-host`'s padding+border were counted as renderable. Fix moved chrome to `.dv-host-term`, leaving `.term-host` zero-chrome.
- e2e native mock is keyed by command NAME only (static) — `list_dir` returns the same fixture for every path. Args-based mocking would need a nativeTransport change if the turns/tree tests need per-path data.
