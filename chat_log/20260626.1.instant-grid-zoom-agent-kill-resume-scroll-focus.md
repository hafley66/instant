- Scroll fix: pty.rs scroll_session(name, up, lines) = `tmux copy-mode -e -t name` then `tmux send-keys -t name -X -N <n> scroll-up|scroll-down`; registered in lib.rs. Shift+wheel (capture phase, stops propagation) → invoke scroll_session. Plain wheel left alone (in-app mouse). Shift+wheel puts you in tmux copy-mode; scroll to bottom (-e) or q to resume typing.

## Key Insights
- xterm-in-tmux has no real scrollback; xterm.scrollLines is a no-op. Wheel scroll = tmux copy-mode. Plain wheel is app-dependent (claude/opencode grab the mouse). Shift+wheel→backend tmux copy-mode is the app-independent fix.
- onTermClosed only fires on genuine dockview panel removal (close/⌘W/kill menu), NOT on location.reload() — a hard reload tears down the page without running React effect cleanups or onDidRemovePanel. So kill-on-close is reload-safe.
- A single requestAnimationFrame is NOT enough to focus a terminal on tab switch — dockview can make the panel visible that same frame (focus on display:none no-ops) and can steal focus after. Need rAF + a short setTimeout retry.
- resumeTabs must be keyed by CWD, not tmux session name (reopen mints a fresh name). And the live pane cwd ≠ the scan's worktree-root path the reopen passes — record under BOTH (use worktreesForPaths to map pane cwd → worktree).
- kill_session removes the pty keyed `s:{name}`; sessionId(name)=`s:${name}` so keys match. open_session passes the launch command as a tmux ARG (ignored on -A reattach), so it does NOT write a newline to the pty.
- macOS accessory app: NSApp/window hide doesn't return focus; `open -a "<name>"` reactivates the prior app with no new TCC prompt.
- TanStack: state.expanded `true` = all-expanded (not persist-friendly); expand by enumerating flatRows ids instead. filterFromLeafRows keeps ancestors of matches.
- enable_mouse sets tmux `mouse on` per-session with 15x60ms retries.
