# Session: instant-json-rx-live-v2-metrics

## Goal
Make Instant's live usage capture execute through automation.v2/JSON-Rx, preserve the existing Rules authoring and dashboard wire interfaces, update the architecture documentation, and leave a resumable verified worktree.

## Current State
Implementation is complete and uncommitted on branch `codex/live-v2-metrics` in `/private/tmp/instant-live-v2-metrics`, based on main commit `1a8e2df`.

Live Claude and ChatGPT netcapture rules now lower into automation.v2 in the extension. Browser responses enter per-rule RxJS Subjects, run through the production JSON-Rx compiler's JSONata project and shared root, and emit the existing DashboardEmission to `/ingest`. Rust persists the common `rulematch` envelope. Rules history and Metrics consume the same database path as before.

Selector and text-node rules retain their direct compact interpreters. Regex-only netcapture rules retain their fallback. Nettcapture rules with `response.extract` and `emit` use v2.

## Problem/Context
Before this change, automation.v2 had a production schema/compiler and tested Claude/Codex definitions, but the live extension still used the v1 extraction function. Metrics could consume v2-shaped envelopes but received live rows from direct Rule JSON execution.

## Solution/Approach
- Extended the v2 project node with `language: "jsonata"` and arbitrary nonempty field expressions.
- Changed project compilation to ordered asynchronous JSONata evaluation with passed/missing/error trace callbacks.
- Preserved page URL and capture timestamp as dashboard provenance.
- Added `extension/src/6_v2Rules.ts` with `ruleToAutomationV2` and `createV2RuleRuntime`.
- Config replacement disposes old subscriptions and creates one v2 runtime per active extract+emit netcapture rule.
- Content-world response messages feed matching v2 source Subjects.
- V2 emissions relay through the service worker with `automationVersion: "automation.v2"` at the raw ingest boundary.
- Empty projections produce diagnostics without empty persisted rows.
- Updated the isolated lab to the same JSONata project semantics.
- Updated the JSON-Rx book, extension README, Metrics plugin README, changelog, and production plan.

## Key Insights
- Rule JSON remains a compact authoring syntax. Runtime lowering can make v2 the execution backbone without migrating persisted rules or changing the Rules table.
- DashboardEmission is the stable join between execution and consumers. Rust persistence and Metrics rendering did not need a second v2-specific path.
- Config revision replacement is the v2 runtime cancellation boundary.
- `concatMap` gives ordered async JSONata projection. There is no explicit queue bound yet.
- Browser ChatGPT capture is the live Codex usage source. The typed native Codex host adapter remains pending and optional.
