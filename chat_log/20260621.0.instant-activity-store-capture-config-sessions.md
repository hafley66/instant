## Solution/Approach
- Unified store: spy.rs absorbed into activity.rs -> one activity.db, events(id,ts,source[browser|os|files|session],kind,app,url,title,text,shot)+idx. Ingest server (tiny_http :8787) writes source='browser'. os shots capped newest 2000 (row+PNG pruned); browser/files/session text = 7-day ring.
- capture.rs: event-driven screencapture off the CGEventTap (click/dblclick/drag-edges/Cmd+C/Cmd+V), 350ms throttle, spawned off tap thread. frontmost app via CGWindowList (lsappinfo ASN lookup returns empty on this macOS, only -pid worked, so switched to CGWindowList kCGWindowOwnerName at layer 0). Skips capture while instant window is focused (WindowFocused Arc<AtomicBool> set by tauri WindowEvent::Focused) so clicking in-app never self-records.
- config.rs: config.json (exclude_sites/files/apps), case-insensitive substring + '*' anchored glob, host_of() so *.bank.com matches chase.bank.com/login despite path. Block-future-only (no purge). Editable in-app via config_set; config_get/reload/open commands.
- fuzzy.ts: subsequence scorer (consecutive/word-start/prefix bonuses, gap penalty). ctxmenu.ts: XP raised menu, contextual per terminal/file-row/activity-row/default, edge-flip, dismiss on outside/Esc/scroll/blur.
- Activity panel (replaces Spy): search box + source chips [all|sessions|browser|screen|files] + event-type chips [all|highlighted|copy/paste|images|files|urls|clicks] + Recording toggle + Clear; table time/src/kind/where/what; preview pane (read_image thumbnail).
- Sessions sidebar = live tmux sessions (list_sessions), attach/resume on click, x kills (new pty.rs kill_session = tmux kill-session). Quick-launch +claude/+opencode buttons + "new shell" input; Spaces creator collapsed in <details>. Session-visit logged source='session' kind='focus' (deduped, suppressed during boot replay via `replaying` flag).
- iTerm2 keybindings via term.attachCustomKeyEventHandler: Opt+L/R = ESC b/f, Opt+Bksp = ESC DEL, Cmd+L/R = ^A/^E, Cmd+Bksp = ^U.
- Extension v0.2.0: content.js DOM events (click/ctrlclick/dblclick/drag/rclick w/ target+mods), background.js tab lifecycle (tabopen via webNavigation.onCreatedNavigationTarget = the "why", tabswitch/tabclose), manifest += webNavigation.
- CSS fixes: chip selected state moved off `.active` to `.act-chips .act-chip.on` (xp.css owns button.active pressed-shadow at specificity 0,2,1, was forcing white-on-pale). Same fix applied to `.toolbar button.active` and base `.tab-active` -> solid accent fill so active panel toggle / session tab reads clearly across skins (P5/AC3 already had own fills).

## Key Insights
- lsappinfo ASN -> name lookup returns empty on Darwin 23.6; only `lsappinfo info -only name -pid` works. CGWindowList (kCGWindowOwnerName at kCGWindowLayer==0, on-screen front-to-back) is the no-prompt native frontmost-app source.
- xp.css `button:not(:disabled).active` is specificity (0,2,1) and sets pressed box-shadow only; any custom `.active` styling for buttons must avoid the class name or out-specify it, else selections render light-on-light.
- Tauri command JS args are camelCase for snake_case Rust params (excludeSites for exclude_sites).
- App data dir: ~/Library/Application Support/com.instant.summon/ (activity.db, config.json, captures/<YYYY-MM-DD>/<ts>-<kind>.png). Old spy.db orphaned.
- "browser stuff" the user saw was os screenshots showing the browser, NOT a sideloaded extension; though by end an extension WAS loaded (claude.ai nav rows appeared) and is live.
- dev watcher (npm run tauri dev, log /tmp/instant-dev.log) rebuilds backend on src-tauri changes; standalone cargo check races it on the package-cache file lock (stale errors in log are mid-edit saves, trust the final Finished+Running).
