## Problem/Context
- Resume failed repeatedly. The breakthrough was the user's insight "tabs have names, use it" + claude's `--session-id <uuid>` flag.
- Root causes found (in order, each fixed):
  1. harness.rs encoded claude project dir as `cwd.replace('/','-')` but claude-code maps EVERY non-alphanumeric char (incl. '.') to '-'. Worktrees live under `.worktrees`, so the dot broke every lookup. (Real but NOT the user's case — they run plain repo roots.)
  2. openTab's resume only fired when `command==null`; every real reopen (⌘⇧T replays original cmd; worktree pre-resolves) passed non-null → bypassed.
  3. My first "fix" (unconditional killed-record override in openTab + widened keys) caused regressions: "new · X" resumed instead of fresh, dupe/wrong tabs from cross-contamination.
  4. autoResume defaults TRUE (state.ts:224, persisted) → every open ran resumeCommand → resumed latest-in-cwd = "opens most recent".
  5. claude's foreground proc shows as its VERSION string ("2.1.193"), never "claude", so AGENT_PROCS never matched → close DETACHED instead of killing → "only newlining, not closing".
  6. harness_session resolves "newest jsonl in cwd", but multiple claude sessions share `/projects/instant` → grabbed a sibling = "random old session". Plus persisted resumeTabs was poison from all prior attempts.

## Solution/Approach (the WORKING design, commit 1bf9519)
- DETERMINISTIC ID: `newAgentLaunch(name, command)` in main.ts — if launching bare claude (no --resume/--session-id/--continue/--from-pr), generate `crypto.randomUUID()`, launch `claude --session-id <uuid>`, record `resumeTabs[name] = {editor:"claude", sessionId:uuid}` AT LAUNCH. We own the id; no probe, no cwd collision. opencode has no launch-with-id flag → falls back to close-time cwd probe.
- KEY BY SESSION NAME (user's insight), not cwd. resumeTabs: Record<sessionName, {editor,sessionId}>.
- openWorktree(fresh): fresh=true → freshSessionName (suffixed) + newAgentLaunch (always new). fresh=false → baseSessionName + resume known record if present (resumeLaunch) else newAgentLaunch. open_session's tmux `-A` reattaches a live base session (ignores the command), so resume cmd only matters when base was killed.
- openWorktreeDefault (double-click) changed true→FALSE = resume-or-new on base.
- showAgentMenu "new · X" / "new shell" stay fresh=true (always new).
- reopenLastTab (⌘⇧T): resume by `resumeTabs[last.name]`, KEEP the record (don't delete) so name→id is stable across repeated reopens.
- exitOrDetachTab: kill detection via `looksLikeAgentProc(proc)` = AGENT_PROCS ∪ /^\d+\.\d+/ (claude version) ∪ includes("opencode"). On close, DON'T clobber an existing resumeTabs[name] (the launch-time owned id is authoritative); probe-record only when no record exists.
- Removed the whole autoResume-on-open path (deleted resumeCommand fn + the "auto-resume latest" menu item). autoResume state field now vestigial.
- POISON CLEAR: state.ts load() one-time `resumeTabsV2` localStorage flag wipes old cwd-keyed resumeTabs.
- harness.rs: encode non-alnum→'-' to match claude (helps real `.worktrees` worktrees).
- ⌘-CLICK OPEN (eb6ab82): main.ts `tokenAt(id,x,y)` maps pointer cell→buffer token (trims wrapping brackets/quotes + trailing punct, keeps `:line:col`); `looksOpenable` gate; capture-phase metaKey mousedown on term host. lib.rs `open_target(target,cwd)`: URL→browser, existing path→default app via /usr/bin/open; `strip_line_suffix` + `resolve_path` (~ + relative vs pane cwd); hides window; logs activity source=session kind=open. plugin-opener was present but `opener:default` lacks allow-open-path, so used /usr/bin/open directly (matches existing reactivate_prev_app pattern).

## Key Insights
- claude CLI: `--session-id <uuid>` launches a NEW session with a chosen id (we own it); `--resume <id>` / `-c` resume. opencode: `--session <id>` / `-c` continue, NO launch-with-id flag. Use --session-id to make resume deterministic instead of probing the filesystem.
- claude's tmux `pane_current_command` is its VERSION ("2.1.193") because it sets process.title; it is NEVER "claude". Any proc-name allowlist must match a version pattern. opencode shows "opencode.exe".
- claude-code project dir encoding = `path.replace(/[^a-zA-Z0-9]/g,'-')` (every non-alnum, incl '.'), not just '/'. Verified on disk: `/Users/.../anim/.claude-worktrees/atlas-fold` → `-Users-...-anim--claude-worktrees-...` (double-dash from '/' + '.').
- Multiple agent sessions can share one cwd (instant-main/-2/-3 all in /projects/instant), so any cwd-based id lookup ("newest jsonl in cwd") is fundamentally ambiguous → must key by something tab-specific (we chose: own the id at launch + key records by session NAME).
- Resume must be ONE explicit gesture, never a side effect of any open, or "new" silently reattaches. Persisted state (autoResume=true, stale resumeTabs) survives ⌘R and poisons behavior; semantics changes need a localStorage migration flag.
- forgetTab runs before the async exitOrDetachTab, and refreshSessions mutates the live list — capture any close-time state synchronously before those.
- open_session passes the launch command as tmux's trailing shell-command arg: runs on fresh create, ignored on `-A` reattach. Multi-word commands ("claude --session-id x") work as a single arg.
- plugin-opener `opener:default` permission does NOT include allow-open-path (scope-gated); shelling to /usr/bin/open sidesteps it.
