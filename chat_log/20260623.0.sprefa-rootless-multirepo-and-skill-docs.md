## Key Insights
- sg/ast scan arity: scan(REPO, REV, GLOB, pathvar, revvar) 5-arg, or
  scan(REPO_set, REV, ...) — "WORK" is the REV not a repo. scan("*","WORK",...)
  fans config repos and ALREADY excludes self when config non-empty.
- type_entity is oxc-backed for TS (TsTypes), syn for Rust, tree-sitter Kotlin.
  Lazy: only files a source rule scanned. gen("path",template) file-sink writes
  markdown, converges (no write if bytes match) -> auto-updates under watch.
- gen path sandboxed to root ("escapes the root"). type_entity row dedup by sym
  -> identical-path repos silently dropped rows pre-fix.
- repo::nearest_git(start) walks ancestors for .git. sprefa git root is
  ~/projects/sprefa (parent of v5), so v5 files derive repo="sprefa".
- BSD sed has no \b; use perl for word-boundary substitutions.
- git root is ~/projects/sprefa (parent) — `git add -A` from v5 sweeps junk
  (chat_log, daemon.pid, stray `a`); stage explicit paths. Commit gate via
  core.hooksPath=.githooks -> `exec dl --check` over .dl/rails.dl.
- Bash backticks in -m commit messages get command-substituted; avoid.
