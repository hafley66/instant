## Key Insights
- `$` helper (main.ts:104) is `document.querySelector(s) as T` — returns null cast, does NOT throw on miss; so `$("#x").onclick=` throws TypeError when #x absent. `document.querySelector` calls in render fns were already null-guarded; the `$(...)` wiring calls were the boot crashers.
- refreshSessions still updates the store (store.set at ~506) BEFORE the DOM-render early-return (513), so the react-table panels (which read store via bridge) keep working with the v1 render code dead.
- renderWorktreesPanel early-returns at `if(!count) return` (#wt-count gone) before reaching the throwing `$("#wt-view")`, so renderFlatTable/renderTree are never reached either — all dead but harmless.
- Component panels (those with PanelDef.component) skip PanelDef.onShow's pool-adoption; their mount useEffect calls bridge.onShow. onShow must be a block-body effect returning nothing (returning the Promise -> React calls it as cleanup -> "destroy is not a function" crash, fixed last session).
