## Key Insights
- TCC (Accessibility/Input-Monitoring/Screen-Recording) keys on the binary's code-signature designated requirement. Ad-hoc/linker-signed debug binary → cdhash changes every rebuild → grant dropped. Stable self-signed cert → constant DR → grants persist. tauri dev execs the binary right after linking, so a linker shim (signs during link) is the only hook that beats the watcher race; tauri dev does NOT use cargo `runner`.
- cargo-audit 0.22 reads config from .cargo/audit.toml, NOT ./audit.toml. --target-os/--target-arch does NOT prune platform deps from a generic Cargo.lock. cargo tree -i <crate> returns empty for Linux-only deps on macOS (proof they're not in the build).
- opencode part types: text/reasoning carry real `text`; tool carries tool name + state.input.command + state.output. reasoning parts can be empty-text. Many assistant messages are step-only (no text) → empty → skipped, so latest_ai_message returns last real turn.
- claude jsonl: each line uuid/parentUuid/sessionId/timestamp/type(user|assistant|system|mode|file-history-snapshot)/message{role,content}/cwd/gitBranch. content = string OR array of {type:text|thinking|tool_use|tool_result}.
- store.set fires listeners only for CHANGED keys; useApp() subscribes with no keys → re-renders on any change; setting aiFavs to a fresh array always triggers.
- dockview activeGroup.element is a valid positioning context for an in-pane overlay (the toast).
- CtxItem requires `action` even for disabled header items (give them a noop).
