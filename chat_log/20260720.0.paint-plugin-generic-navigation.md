Implement generic plugin panel lifecycle support for Paint and Markdown instances, preserve Paint layer state across close/reopen, and make recent-file navigation prompt on unsaved changes while replacing existing layers.

## Solution/Approach
- Generic `Plugin.instances` and instance lookup/component registration support.
- Dockview captures closed fixed and dynamic panels generically and reopens them with parameters.
- Paint uses per-panel state and per-panel localStorage quicksave/SVG slots.
- Clean Paint close preserves quicksave for Cmd+Shift+T revival; confirmed dirty discard clears it.
- Recent, toolbar, and Enter loads use the dirty-navigation prompt. Save is attempted on confirmation; discard clears the snapshot.
- Image and SVG loads reset the miniPaint layer collection before loading, preventing additive recent loads.
- Restore/reopen loads bypass the navigation prompt.
- Browser E2E native transport stub supports Chromium testing.
- Playwright E2E creates three miniPaint layers, saves, sends Cmd+W, sends Cmd+Shift+T, verifies three layers revive, then loads a recent file and verifies replacement leaves one layer.
