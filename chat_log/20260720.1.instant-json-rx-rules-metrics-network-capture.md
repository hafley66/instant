## Solution/Approach
### Extension capture
- `extension/src/inject.ts` wraps page `fetch` and XHR at `document_start` in MAIN world.
- Isolated `content.ts` writes method/URL capture patterns to `data-ext-netcapture` and receives `window.postMessage` responses.
- `inject.ts` buffers up to 8 usage-like responses before config and replays entries after the attribute mutation.
- Replay now calls `post("response", method, url, { body, status })`.
- `background.ts` webRequest diagnostics are scoped to the Claude usage endpoint.
- `3_extract.ts` runs JSONata field expressions and emits structured pass/filtered/missing/error traces according to rule diagnostics.

### Rust ingestion
- Explicit `rulematch` events bypass generic browser activity `exclude_sites`; the enabled rule is the capture authorization.
- Generic browser activity still uses site exclusions.
- `/diagnostics` exposes `netcapture.*` and `rule.trace` records.

### JSON-Rx
- Public types remain in `src/lib/json-rx/0_types.ts`.
- State updates support replace, JSON Patch, and JSON Pointer set.
- `runMachine` lowers machines to RxJS `scan`.
- Partitioned machines use `groupBy + mergeMap`.
- RxJS operators remain direct exports for concurrency/cancellation semantics.
- JSON Logic handles predicate/guard expressions; JSONata handles projection.
- Catalog references resolve reusable machine and flow definitions.
- Metrics polling uses `timer -> exhaustMap -> native invoke -> Event -> runMachine -> State`.

### Dashboard interface
The generic boundary is `(stream, JSON Schema, timestamped records, provenance)` using the existing rulematch envelope:
- `stream`: stable dataset/dashboard identity
- `schema`: schema for one record and standard rendering annotations
- `matches`: one or more extracted records
- `ts`, `ruleId`, `url`: provenance
Vega-Lite is the visualization grammar. TreeTable remains the table renderer. Later explicit dashboard definitions can carry a Vega-Lite spec or named spec reference without inventing another chart grammar.

### Future network recording/export
Documented architecture:
- Chrome DevTools `chrome.devtools.network` supplies HAR entries and response content for explicit record/select workflows.
- HAR 1.2 is immutable observation storage.
- `har-to-openapi` can infer OpenAPI 3.1.
- OpenAPI Overlay 1.1 stores JSONPath-targeted annotations separately.
- Arazzo 1.1 stores call sequences/dependencies/workflows.
- Bruno CLI imports OpenAPI into OpenCollection, `.bru`, or JSON, so OpenAPI export covers Bruno.
- Capture adapters consume normalized HAR plus local tags/redactions/path-parameter selections/joins and emit target formats.
- External joins should produce ordinary schema-governed dashboard records.

### Future browser actions
Use protocol-shaped JSON-Rx effects based on WebDriver BiDi names where applicable:
```json
{
  "op": "browsingContext.reload",
  "input": {
    "target": { "match": { "url": "^https://claude\\.ai/" }, "cardinality": "one" },
    "ignoreCache": false,
    "wait": "complete"
  }
}
```
The JSON-Rx machine emits an Effect. A browser effect interpreter resolves logical targets to extension tab/context IDs, invokes `chrome.tabs.reload` or `chrome.scripting.executeScript`, and emits a correlated success/error Event with `causationId` back into the machine. Keep logical target selection separate from action params because extension tab IDs are ephemeral.

Two trigger adapters were identified:
- `tab.reload` / `browsingContext.reload` after a matched tab satisfies an inactivity predicate.
- `request.replay` for an explicitly captured sanitized request template executed in MAIN world with the page's credentials.
Use one active trigger per rule (`exhaustMap`), minimum intervals, jitter, `Retry-After`, exponential backoff, and stop on 401/403/429. Explicit request replay should omit copied cookies/auth headers and let same-origin page fetch attach current credentials.

## Key Insights
- Extension execution has MAIN page, isolated content, and service-worker worlds. The DOM attribute plus `postMessage` bridges MAIN and isolated worlds.
- Configuration arrival and response arrival are concurrent; E2E must force both orders.
- `webRequest` can provide metadata but ordinary MV3 webRequest does not provide response bodies. DevTools Network or debugger/CDP can.
- `chrome.devtools.network` maps directly to HAR and fits an explicit record/select interaction, but requires DevTools to be open.
- `chrome.debugger`/CDP supports background body capture with a broader permission/attachment surface.
- WebDriver BiDi supplies stable serializable names such as `browsingContext.reload` and correlated navigation IDs. An extension adapter can implement a subset.
- Logical context selectors belong in Instant configuration; concrete Chrome tab IDs belong in interpreter state.
- Effects are descriptions and effect results are events. This preserves machine replayability and correlation.
- JSON Schema governs record data; Vega-Lite governs visualization; OpenAPI governs HTTP contracts; Overlay governs annotations; Arazzo governs API workflows; HAR governs observed traffic.
- Raw responses stay in the page/extension unless recording is explicitly enabled. Current rulematch transport sends extracted fields to `127.0.0.1` only.
